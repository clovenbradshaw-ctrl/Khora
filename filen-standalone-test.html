Standalone Filen Login/Data Test

Use this page to quickly test the UX flow (register/login/add/view) without changing the main Khora app. It includes an SDK diagnostics check and a local fallback mode so you can verify the UI even when the Filen browser SDK fails.

1) SDK diagnostics

Checks whether the Filen browser SDK can load directly from CDN in this environment.

If this fails with module specifiers like os/path, use the Vite/polyfill setup before deploying to GitHub Pages.
Ready.

2) Login + session (local harness)

Session status: not logged in

This is intentionally a test harness. It stores a pseudo-session in localStorage and does not call Filen.

3) Add/View data (local harness)

Records are saved to localStorage["khora-filen-test-records"].

    How to run a real Filen test on GitHub Pages

    1. Create a Vite app and install @filen/sdk + vite-plugin-node-polyfills.
    2. Configure polyfills in vite.config.js and set base to your repo path.
    3. Deploy the dist/ output to GitHub Pages (gh-pages branch).

    See: filen.io
